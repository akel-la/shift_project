# ИМПОРТ:

Импорт в виде from ... import * гарантирует, что будут импортированы все модели, и при изменении в моделях не нужно будет менять импорты в нескольких разных местах.

# RUN_ASYNC_MIGRATION:

```python
async def run_async_migrations():
    url = config.get_main_option("sqlalchemy.url")
    # Асинхронная работа - требует асинхронный движок:
    # poolclass = pool.NullPool - это отключение пула соединений
    # для миграций, так как миграции должны выполняться быстро и не
    # должны удерживать соединения:
    connectable = create_async_engine(url, poolclass=pool.NullPool)

    async with connectable.connect() as connection:
        # Движок асинхронный, но для миграций нужен синхронный подход.
        # Миграции - всегда должны выполняться одним актором, без параллельных SQL запросов и переключений, иначе могут быть проблемы.
        # Пример - несколько процессов одновременно шлют SQL запросы-миграции в одну СУБД. Это может привести к поломке.
        # Но для синхронной работы - нужен синхронный движок.
        # Чтобы не создавать лишних движков -
        # запускаем асинхронный движок в синхронном режиме:
        await connection.run_sync(do_run_migrations)

    # Закрытие движка, освобождение ресурсов:
    await connectable.dispose()
```
