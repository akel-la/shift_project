Все исключения, которые вручную написаны для этого проекта (кастомные исключения) - имеют общий суперкласс:

Пример:
```python
class ServiceError(Exception):
    """Базовое исключение приложения."""
    def __init__(self, message: str = "Внутренняя ошибка"):
        self.message = message

class NotFoundError(AppError):
    """Ресурс не найден."""

class ConflictError(AppError):
    """Конфликт целостности данных."""

class UnauthorizedError(AppError):
    """Ошибка аутентификации."""

class ForbiddenError(AppError):
    """Доступ запрещён."""
```

Такие исключения возбуждаются только на сервисном уровне.

# ИСКЛЮЧЕНИЯ И СЛОИ:

Репозитории - могут использовать только ошибки ORM библиотек.

Сервисный слой - перехватывают исключения из репозиториев, и возбуждают кастомные исключения.

API - подключенный к app = FastAPI(...) обработчик исключений перехватывает кастомные исключения преобразует их в ответы из стандартного набора для REST API (500, 404, 403 и тому подобное).
