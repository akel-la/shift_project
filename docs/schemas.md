# BASESCHEMA:

Все схемы (в конечном счете) наследуются от схемы:
```python
class BaseSchema(BaseModel):
    model_config = ConfigDict(
        extra="forbid",
        from_attributes=True,
    )
```

extra="forbid" - если клиент пришлет поле, которого нет в схеме (к примеру, "role" = "admin"), то FastAPI:

1. Прервет выполнение запроса.
2. Вернет ошибку 422, чтобы пользователь знал, что он делает неправильный запрос.


from_attributes=True - связка между Pydantic и SQLAlchemy. Позволяет создавать экземпляры Pydantic схем из объектов ORM, без этой настройки response_model не сможет работать.

# Response и CreateResponse:

CreateResponse - схема объекта, который возвращаем при успешно выполненном CREATE запросе.
Response - то, что возвращаем при GET запросе.

Response наследуется от CreateResponse.
Обе схемы содержат только те поля, которые можно делать публичными.
У некоторых сущностей эти две схемы могут быть одинаковыми, но они могут начать различаться при расширении проекта.

# ПРОВЕРКИ:

Уровень схем проверят только то, для чего достаточно присланных данных (не нужно делать запросы в БД, вызывать сложную бизнес логику и тому подобное). Соответствия типам, длинна коллекций, время начала раньше времени окончания и тому подобное.

# PYTHON 3.14:

При переходе на python 3.14, могут быть проблемы из-за изменения механики работы аннотаций, особенно у схем, которые в качестве своих полей содержат другие схемы.
См. сообщения об ошибках, а так же:

```python
from __future__ import annotations
```

и

```python
New_Schema.model_rebuild()
```
